Dependent files

  • /canvas/oembed/index.php
    • This formats the json based on the html it grabs from $_GET['html'] which I am sending to Canvas as part of the url parameter. This is not exactly what LTI expects. In general, they are expecting the oembed endpoint to point to a location that is storing the json, but I did not want to complicate my life with another data storage system. It is a good idea to keep the amount of data in the html relatively short. I am doing that by returning a shell to Canvas, with the expectation that users will create the bulk of their content after returning to Canvas.
  • /canvas/editorBtns/editor_files
    • neither of these files is necessary for the buttons to work. They serve only to provide a preview in the case of the glossary. I should probably remove that option. It is unnecessary, and quite possibly an imperfect match to what will be shown back in Canvas. Still, it's a pretty slick feature.
