This was originally created as a standalone app for use in the Admin menu. Then I added the option for installation as a course menu item, using a temporary API auth token, thinking I would share it more broadly. Then I decided to back off on the whole token dance, and delivered it as a pre-installed navigation map for all courses with a permanent token. Finally I decided to wrap it into Utilities.

Theoretically this app is still installable as a standalone app, using either of the configuration xml files in this folder, but this usage has not been tested by me. It is perfectly possible that the standalone app is in use somewhere, and in any case, the launch url in utilities includes some of the files here, so this folder should not be touched.

PHP Functions

  • sortByName($a, $b) 
    •  * sort function used by uasort to arrange users by their ['sortable_name']
  • byname($a,$b) 
    •  * another sort function, this one sorts an indexed array alphabetically based on the value
    of the [0] index
  • sortByDate($a,$b) 
    •  * another uasort sorting function. This one sorts on ['due_at']

Javascript Functions

  • parseurl() 
    •  * pulls the course and asst ids out of the assignment url

Session Variables

  • $_SESSION['input']
    • The value (paste | pulldown) of this determines whether users select an assignment from a dropdown list or paste in a url. The version running in Utilities only supports the latter. Standalone versions branch based on which config xml you use. If using the config.xml.php send input=paste|pulldown as a get string. Again, this has not been tested since I switched to Utilities.
